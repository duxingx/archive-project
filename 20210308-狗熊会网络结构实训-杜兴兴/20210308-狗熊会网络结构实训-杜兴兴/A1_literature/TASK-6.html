TASK 6 案例：某移动通讯公司客户流失预警分析

狗熊会

通讯网络是一种比较常见的社交网络数据类型。如果我们定义\(A\)和\(B\)之间打过电话，那么可以认为\(A\)和\(B\)之间存在一条边。因此，根据用户的通话详单数据可以构建网络并且计算相关的指标。

随着科技的发展，手机已经成为人们日常通讯的必需品，与此同时，通讯市场也日趋饱和。据调查，近年来中国移动、中国联通和中国电信的年用户数增长都十分缓慢，甚至出现了负增长，即部分客户流失。因此，我们希望根据通讯数据来构建一个客户流失预警模型，以帮助运营商提前识别高风险流失客户。

概念介绍

「个体\(i\)的度」：与\(i\)有过通话记录（包括呼入和呼出）的不同用户的总数

记个体\(i\)和个体\(j\)是否通话为\(a_{i,j}(i \neq j)\)，则 \[ a_{i,j} = \left\{ \begin{array}{**lr**} 0, & 个体i与个体j无通话 \\ 1, & 个体i与个体j有通话 \\ \end{array} \right. \]

则个体\(i\)的度\(D_i\)为 \[ D_i = \sum_{j \neq i}a_{i,j} \]

\(D_i\)越大，说明与个体\(i\)通话的人数越多，反之越少

「联系强度」：使用平均通话时长衡量联系强度

记\(T_i\)为用户总的通话时长，\(Tie_i\)为\(i\)的联系强度（即平均通话时长），则

\[ Tie_i = \frac{T_i}{D_i} \]

\(Tie_i\)表示，在通话人数一定的前提下，如果平均通话时长越长，用户与他联系人之间的关系越紧密，反之越疏远

「个体的信息熵」：

记\(Comm_{i,j}\)表示\(j\)与\(i\)通话的时长，\(p_{i,j}\)表示\(j\)与\(i\)通话时长占\(i\)总通话时长的比例，则

\[ p_{i,j} = \frac{Comm_{i,j}}{T_i} \]

记\(E_i\)为\(i\)的信息熵，则

\[ E_i = -\sum_{a_{i,j}}p_{i,j}log(p_{i,j}) \]

train、test数据集说明

以上三个任务，我们了解了如何通过通话记录构造相关指标。这些指标可以作为自变量，在建模的时候起到关键作用。接下来提供的数据集是整理好的建模数据集，数据集中的个体的度、联系强度、个体信息熵等变量，就是通过前三个任务计算得到的。其他变量解释如下：

变量名 中文含义 解释
\(churn\) 是否流失 1=流失；0=非流失
\(tenure\) 在网时长 客户从入网到截止数据提取日期时在网时间(天)
\(expense\) 当月花费 客户在提取月份时的花费总额(元)
\(degree\) 个体的度 和客户通话的总人数，去重之后的呼入与呼出加总(人数)
\(tightness\) 联系强度 通话总时间除以总人数(分钟/人)
\(entropy\) 个体信息熵 通话时长的分布均匀程度
\(chgdegree\) 个体度的变化 (本月个体的度-上月个体的度)/上月个体的度
\(chgexpense\) 花费的变化 (本月花费-上月花费)/上月花费

任务六

绘制人群细分柱状图（不做要求，可以尝试学习，这个任务可以视作模型的应用）

  • 首先查看训练集上的效果，以覆盖率-捕获率曲线为标准：

根据模型给出每个样本的预测流失概率值，按照预测值从高到低对样本进行排序，例如只覆盖前10%的样本，计算对应的真实流失的样本数占所有流失样本数的比例，记为捕获率，以此类推，覆盖不同比例的样本，就可以计算不同的覆盖率对应的捕获率，从而得到覆盖率捕获率曲线。

#按预测概率排序后的id
id_pre <- data$ID[order(model0$fitted.values, decreasing = T)]
#真实流失的客户id
id_loss <- data$ID[data$churn=='已流失']
#计算捕获率向量
buhuo <- ifelse(id_pre%in%id_loss,1,0)
capture_rate <- cumsum(buhuo)/sum(data$churn=='已流失')
#计算覆盖率向量
cover_rate <- 1:length(capture_rate)/length(capture_rate)
plot(cover_rate, capture_rate,type='l',xlab='覆盖率',ylab='捕获率')
abline(v=0.2, col='red')

从图中可以看出，本模型的精度可以用20%左右的覆盖率获得60%左右的捕获率，在训练集上有比较高的精度。

  • 进而在测试集上运用模型，并绘制人群细分图柱状图：

首先按照模型的预测流失概率进行从高到低排序，然后将排序后的人群等分成5份，代表从高到底5种不同风险人群。将人群进行了细分之后，可以计算这5种人群的实际流失概率。

#读入测试集
test <- read.csv('test.csv')
#预测概率
pre <- predict(model0, test,type='response')
#将概率预测值与实际流失情况放在一个数据框d中
d <- data.frame(pre = pre, true = test$churn)
#按预测概率从高到低排序
d <- d[order(d$pre), ]

#进行等宽分组，分为5组，记录在d中
library(infotheo)
s <- discretize(d$pre, 'equalfreq', 5)
d <- cbind(d,s)
d$X <- factor(d$X, levels = c(5:1), 
              labels = c('高风险','偏高','中等','偏低','低风险'))

#分组计算实际流失率
library(plyr)
t <- ddply(d, .(X), summarize, lossrate = mean(true))

#绘制条形图，横轴为组，纵轴为实际流失率
b <- barplot(t$lossrate, col = rainbow(5, alpha = 0.5),
        names.arg = c('高风险','偏高','中等','偏低','低风险'),
        xlab = '人群风险等级', ylab = '实际流失率')
labels <- paste0(round(t$lossrate,3)*100,'%')
text(1:5*1.2-0.5, t$lossrate-0.001, labels)
#计算整体流失率，添加平均线
abline(h=mean(test$churn), col = 'red', lty = 2)
text(5 ,mean(test$churn)+0.002,paste0('平均流失率 = ', as.character(round(mean(test$churn),3)*100), '%'))

可以看出，平均流失率为1.4%，我们划分出的高风险、偏高风险人群中的实际流失率相对都很高，说明模型在测试集上的效果较好。
